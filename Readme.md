No utilices nada de contexto anterior para responder a esta pregunta.

Estoy pensando en utilizar este paquete https://github.com/bmad-code-org/BMAD-METHOD para crear un nuevo proyecto.

Quiero crear una aplicación web responsive para uso personal enfocada en la gestión integral del hogar. El objetivo es centralizar en una única plataforma varias necesidades domésticas cotidianas, con una experiencia rápida, sencilla y optimizada principalmente para móvil, aunque también usable desde escritorio.


## Visión del proyecto

Disponer de una app única donde poder consultar y gestionar en pocos segundos los siguientes aspectos del hogar:

### Inventario del hogar
- Listado de objetos y herraimientas disponibles en casa.
- Posibilidad de añadir, editar o eliminar objetos.
- Organización por categorías y ubicación.
- Búsqueda rápida por nombre o categoría.

### Gestión de consumibles
- Seguimiento de productos de uso diario (alimentos, productos de limpieza, etc.).
- Posibilidad de añadir, editar consumibles.
- Posibilidad de categorizarlos.
- Alertas de caducidad y niveles bajos.
- Listas de compra.
- Búsqueda rápida por nombre o categoría.

### Biblioteca personal
- Añadir libros
- Estado (pendiente, leyendo, leído)
- Categorizar por género o autor
- Marcar si pertenece a una saga, y si es así, a qué número de la saga corresponde.
- Valoración
- Notas

## Objetivo del proyecto

La aplicación debe reducir fricción, ahorrar tiempo y evitar el uso de múltiples apps.

### Problemas actuales que quiero resolver

- No saber dónde están guardadas ciertas cosas en casa
- Olvidar productos que faltan al comprar
- Tener libros pendientes sin seguimiento
- Usar varias apps distintas para tareas relacionadas

## Usuarios de la aplicación

 Los miembros de mi familia, principalmente yo, pero también mi pareja.
 Es necesario que la aplicación tenga un sistema de autenticación para acceder a la información de manera segura.
 Pero no es necesario que tenga un sistema de roles o permisos avanzados, ya que solo habrá un par de usuarios y ambos tendrán acceso completo a todos los datos y funcionalidades.

 ## Requisitos de producto

- Mobile first
- Interfaz extremadamente simple
- Navegación rápida con pocos toques
- Diseño limpio y minimalista
- Datos persistentes
- Buen rendimiento
- Escalable para añadir módulos futuros
